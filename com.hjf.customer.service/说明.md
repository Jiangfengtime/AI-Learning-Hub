# 多轮对话方式

### 传统Pipeline架构（NLU→DM→NLG）

核心设计思想

1. 用LLM理解用户意图
2. 用结构化命令控制对话流程
3. 用可编程动作执行流程

核心三元设计

| 要素 | 英文    | 职责           | 实现                      |
| ---- | ------- | -------------- | ------------------------- |
| C    | Context | 提供完整上下文 | Tracker + 对话历史 + 槽位 |
| A    | Action  | 执行具体动作   | 可插拔动作体系            |
| M    | Memory  | 记忆对话状态   | 对话栈 + Flow历史         |
